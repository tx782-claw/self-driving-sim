## 🎯 P3-A: Ego-motion 补偿 (Day 1-5)

### 问题诊断
当前 EKF 预测只用 `dt=0.05` 常数,**未用 IMU 推算自车运动**。
- 高速 (60 m/s) 场景下,自车 1 帧移动 3m,而目标在传感器坐标系下"看似"移动了额外 3m
- 跟踪器把"自车运动"误判为"目标运动",导致速度估计偏移
- v0.2.2 weather 文档明确指出此问题但未修复

### 任务清单

#### P3-A.1 重构 EKFTrack 接口 (Day 1-2)
**文件**: `fusion/ekf.py`
**改动**:
- `predict(self, dt=None, ego_motion=None)` 新增参数
- `update()` 不动 (EKF 接口稳定)
- 文档化接口契约

**验收**: 现有 7 个 unit test 全部通过,无破坏性改动

#### P3-A.2 IMU 预测器 (Day 2-3)
**文件**: 新建 `fusion/imu_predict.py`
**功能**:
- 输入: `IMU` 检测 (加速度 + 角速度)
- 输出: 1 帧自车运动 `ego_motion = {delta_position, delta_velocity, delta_yaw}`
- 用 `scipy.integrate` 或简单梯形积分

**代码骨架**:
```python
class IMUEgoPredictor:
    def __init__(self, dt: float = 0.05):
        self.dt = dt
        self.last_accel = None
        self.last_gyro = None
        self.velocity = np.zeros(3)  # 积分得到

    def update(self, imu_det: Detection) -> np.ndarray:
        """返回 ego_motion dict"""
        if self.last_accel is None:
            self.last_accel = imu_det.position  # 加速度
            return {'delta_position': np.zeros(3), 'delta_velocity': np.zeros(3)}

        # 梯形积分
        delta_v = 0.5 * (self.last_accel + imu_det.position) * self.dt
        self.velocity += delta_v
        delta_p = self.velocity * self.dt

        self.last_accel = imu_det.position
        return {
            'delta_position': delta_p,
            'delta_velocity': delta_v,
        }
```

#### P3-A.3 Tracker 接入 (Day 3-4)
**文件**: `fusion/tracker.py`
**改动**:
- `MultiObjectTracker.update()` 在调用 track.update 之前,先调 `predict(dt, ego_motion)`
- `core/simulator.py` 把 IMU 数据传给 tracker

#### P3-A.4 测试 (Day 4-5)
**文件**: 新建 `tests/unit/test_ego_motion.py`
**测试**:
- 静态自车 (v=0) + 静态目标 → 无补偿时也应无漂移 (baseline)
- 高速自车 (v=60 m/s) + 静止目标 → 补偿后位置 RMSE 应 < 0.5m
- 50 帧仿真,无 ID switch

**验收**: High-Speed 场景位置 RMSE 降低 ≥ 10%

## 🎯 P3-B: GNNS 数据关联 (Day 6-12)

### 问题诊断
当前 Hungarian 是**欧氏距离门限 + 贪心匹配**:
- 不考虑 track 协方差大小(不确定度)
- 密集场景下相邻 track 互相\"抢\"检测,ID Switches 偏高
- v0.3 在 Dense Highway 24 车场景下 ID Switches ~30

### 任务清单

#### P3-B.1 Mahalanobis 距离工具 (Day 6-7)
**文件**: `fusion/association.py`
**新增**:
- `_mahal_distance_sq(det_pos, trk_x, trk_P)` 函数
- `_chi2_gate(d2, df=3, confidence=0.95)` 函数 (返回 True/False)
- chi2 95% 分位 = 7.815, 99% 分位 = 11.34

**复用**: jpda.py 已经有 `_mahal_distance`,可以 import 或复制

#### P3-B.2 GNNS 关联器 (Day 7-9)
**文件**: 新建 `fusion/gnns.py`
**算法**:
1. 对每个 (det, trk) 算 Mahalanobis 距离
2. 过滤: d² < chi2_gate(confidence) 留下
3. 在过滤后的候选对上跑 Hungarian (scipy.optimize.linear_sum_assignment)
4. 返回 (matched, unmatched_dets, unmatched_trks)

**接口**:
```python

    """GNNS = Global Nearest Neighbour Standard filter
    Args:
        gate_chi2: 3D chi2 95% 分位, 越大越宽松
    """
```

#### P3-B.3 Tracker 集成 (Day 9-10)
**文件**: `fusion/tracker.py`
**改动**:
- `association_mode='gnns'` 开关
- 替换 `hungarian_associate` → `gnns_associate`
- WebUI 加 radio 按钮 (P3-F 验收时)

#### P3-B.4 测试 (Day 10-12)

## 🎯 P3-C: IMM 3 模型 (CV + CA + CTRV) (Day 13-19)

### 问题诊断
当前 IMM 只有 CV + CA:
- 弯道 (Junction 场景) CTRV 应更优
- 摩托/行人随机机动 Singer 模型更好 (但成本高,先不做)
- 3 模型是\"性价比甜点\"

### 任务清单

#### P3-C.1 CTRV 模型 (Day 13-15)
**文件**: 新建 `fusion/ctrv.py`
**状态**: `[x, y, v, yaw, yaw_rate]` (5 维)
**运动方程** (线性化,因 yaw_rate 与 v 耦合是非线性):
```python
def f_ctrv(x, dt):
    px, py, v, yaw, yaw_rate = x
    if abs(yaw_rate) > 1e-6:
        px_new = px + v/yaw_rate * (sin(yaw + yaw_rate*dt) - sin(yaw))
        py_new = py + v/yaw_rate * (cos(yaw) - cos(yaw + yaw_rate*dt))
    else:
        px_new = px + v * cos(yaw) * dt
        py_new = py + v * sin(yaw) * dt
    return np.array([px_new, py_new, v, yaw + yaw_rate*dt, yaw_rate])
```
**实现**: 用 UKF 框架 (无需求 Jacobian) — 复用现有 `fusion/ukf.py` 的 sigma_points/unscented_transform

#### P3-C.2 扩展 IMM (Day 15-17)

## 🎯 P3-D: IEKF 升级 (Day 20-24)

### 问题诊断
EKF update 是一次性线性化,在远距/低信噪比时精度差。
- IEKF 多次重线性化 (Gauss-Newton),3-5 次即可收敛
- 代码改动小,主要包装现有 update

### 任务清单

#### P3-D.1 IEKF 实现 (Day 20-22)
**文件**: 新建 `fusion/iekf.py`
**算法**:
```python
class IEKFTrack(EKFTrack):
    def update(self, det_pos, sensor_id, timestamp,
               det_confidence=0.5, max_iter=3, tol=1e-3):
        for i in range(max_iter):
            z_pred = self.kf.H @ self.kf.x  # 预测
            residual = det_pos - z_pred
            # EKF update
            self.kf.update(np.array(residual + z_pred))  # 简化
            if np.linalg.norm(residual) < tol:
                break
        # 更新元数据 (age/hits/miss_streak)
        self._update_metadata(det_pos, sensor_id, timestamp, det_confidence)
```

**注意**: filterpy KalmanFilter 没有 IEKF 模式,需手动控制 predict/update 循环

#### P3-D.2 测试 (Day 22-24)
**文件**: 新建 `tests/unit/test_iekf.py`
**场景**:
- 远距目标 (>50m) — LiDAR 噪声 σ=0.3m
- 强非线性场景 — Camera 观测
- 收敛速度 — 应在 3 次迭代内

**验收**: 远距目标位置 RMSE 降低 ≥ 10%
